# 설계 전환 계획서 — A(핸디캡 vs 풀) → B(NoQ군 vs Q군 맞대결)

**과목:** CS377 강화학습 · **Team 1**
**목적:** 현재 구현된 *비대칭 핸디캡(30 vs 39)* 설계를 *대칭 맞대결(30 vs 30)* 로 전환한다. 후자는 floor effect가 없고, 각 매치업이 "퀸 vs 9점 묶음"을 깨끗이 격리하며, 동일 강도로 돌릴 수 있다.

---

## 0. 무엇이 바뀌나 (한눈에)

| 항목 | A (현재) | B (전환 후) |
|------|----------|-------------|
| 한 게임 구성 | 9점 뺀 쪽 vs **풀(39점)** 상대 | **NoQ군(30) vs Q군(30)** 맞대결 |
| 한쪽 재료 | 30 | 30 |
| 다른쪽 재료 | 39 | 30 |
| 비교 방식 | 패턴별 *풀 대비* 승률(간접) | 매치업 *직접* 승률 |
| Floor effect | 있음 → 강도 비대칭 필요 | **없음 → 동일 강도 가능** |
| 동기 | 레이팅 핸디캡 공정성 | 집중 vs 분산 직접 듀얼 |
| 격리도 | 9점이 풀 대비 빠짐 | **퀸 vs 묶음을 정확히 격리** |

---

## 1. 핵심 불변식 — 왜 B가 깨끗한가

- NoQ군 = `Full − Queen`  (퀸만 제거, **고정**)
- Q군 = `Full − Bundle`  (9점 묶음 제거, 묶음은 가변)
- 따라서 **NoQ군 − Q군 = Bundle − Queen**

두 군대는 공통 코어(`Full − Queen − Bundle`)를 똑같이 갖고, NoQ군은 그 위에 `{묶음}`, Q군은 `{퀸}`만 더 가진다. 즉 매 매치업이 *다른 변수 없이* **"퀸(9) vs 그 묶음(9)"** 을 격리해 묻는다. (예: 묶음=R+B+P → 공통 `1R,1B,2N,7P` + NoQ는 R+B+P, Q는 Queen.)

> 이 불변식은 코드 검증의 핵심이 된다(§7의 isolation 테스트).

---

## 2. RQ / Contribution 재정의

**동기 재서술:** "레이팅 핸디캡 공정성" → **"고정 30점 예산에서, 9점을 퀸에 집중하는 배분과 분산하는 배분 중 near-optimal 플레이에서 무엇이 더 강한가."**

**RQ.**
- **Main:** 퀸을 둔 군대(Q)가 퀸 없이 분산한 군대(NoQ)를 이기는가? (집중 ≷ 분산)
- **Sub-1:** 분산의 정도(묶음 구성)에 따라 결과가 어떻게 변하는가?
- **Sub-2(가산성):** **모든 9점 묶음에 대해 Q군의 승률이 동일(≈0.5)한가?** 동일 → "9점은 어떻게 분산해도 퀸과 등가"(가산성 성립). 다름 → 퀸 대비 묶음별 *유효가치 랭킹*이 드러남.
- **Sub-3:** 흑/백(선·후수)에 따른 차이.

**Contribution (재정의):**
> per-model 학습 교란 없이 단일 net으로, 표준 칸 위에서 "퀸 vs 9점 묶음"을 깨끗이 격리해 비교함으로써, **고정 예산의 집중 배분이 분산 배분을 능가하는지 / 표준 점수 체계의 가산성이 near-optimal 플레이에서 성립하는지**를 검증한다. 배치 자유도 0, 동일 강도로 측정 가능.

---

## 3. 게임 정의 변경 (포지션 생성)

표준 시작 위치에서 **양쪽 모두 제거**:
- **NoQ side** = 퀸 제거 → `{2R,2B,2N,8P}` (30). 고정.
- **Q side** = 묶음 제거 → `Full − Bundle` (30). 묶음은 §4 config.
- 각 매치업 = (NoQ side, Q side, 색 배정). **색 균형**: NoQ-백 / NoQ-흑 양쪽 실행(두 군대가 비대칭이라 선수 이점이 결과에 섞임).
- 제거할 **구체 칸**을 양쪽 모두 고정·정당화(중앙/가장자리 폰, 라이트/다크 비숍, 어느 룩 등).

생성기는 두 측 제거를 적용한 뒤 단일 FEN을 출력. (양측 합 30 assert + 칸에 기대 기물 존재 검증 유지.)

## 5. 분석 모델 변경

1. **per-bundle 승률** — 매치업(NoQ vs Q-bundle)별 Q side 점수(승+0.5무)를 **Dirichlet-multinomial**(W/D/L)로 추정, 95% CrI. 색 균형으로 선수 이점 상쇄.
2. **가산성 검정(Main/Sub-2)** — Q side 점수가 *모든 묶음*에서 0.5(=균형)로 통계적으로 동일한가. 동일 → 가산성 성립; 다름 → 비가산 + 묶음 랭킹. 묶음 이질성을 정직히 다루려면 hierarchical.
3. **유효가치 직접 해석** — 각 매치업이 "퀸 vs 묶음"이라, Q side >0.5면 *그 위치에서* 퀸 > 묶음. 묶음별 정렬 = 퀸 대비 상대가치 순위. (회귀는 선택; 직접 승률이 더 깨끗.)
4. **흑백 비대칭(Sub-3)** — Q side를 백/흑 양쪽으로 → 색별 점수 차.

## 6. 변하지 않는 것 (재사용)

- **P0/P1 수정**(canonical·히스토리·흑 승급 등)은 그대로 필요 — B도 표준 체스다.
- **체스 정확성 게이트**(미러 불변성, 흑이 정상 플레이) — 여전히 필수. (TTT 통과는 이걸 증명하지 못함.)
- **AlphaZero 파이프라인**(배치·멀티GPU), **lc0 Track B**, **단일 net 통제 비교** — 전부 재사용. 변경은 *시작 위치 생성*과 *분석 해석*에 국한.

---

## 7. 검증

- **isolation 불변식 테스트(신규·핵심)**: 각 묶음에 대해 `material(NoQ) − material(Q) == material(Bundle) − material(Queen)` 성립. "퀸 vs 묶음" 격리를 코드로 못박음.
- **포지션 생성 테스트**: 양측 합 정확히 30; 시작 위치 합법; 양 색 생성; 제거 칸이 의도대로.
- **체스 정확성 게이트(재사용)**: 미러 불변성, 소규모 체스 run에서 흑·백 모두 합리적 + lc0/통념 sanity.
- **파일럿 변별성**: 각 매치업이 변별 구간(대략 0.3~0.7)에 드는지. ⚠️ **폰-heavy 묶음(B+6P, N+6P)** 은 폰 6개 차이라 한쪽으로 쏠려 floor가 재발할 수 있음 → 확인 후 별도 취급.

## 9. 주의점 / 결정할 것

- **교차-묶음 비교의 맥락 차이(가벼운 caveat):** 매치업마다 공통 코어가 달라지므로(예: vs R+B+P는 코어 `1R1B2N7P`, vs B+B+N은 `2R1N8P`), *per-매치업* "퀸 vs 묶음"은 깨끗하지만, *묶음 간 랭킹*은 "묶음 가치 + 주변 코어 맥락"이 약간 섞인다. 해석 시 명시.
- **제거 칸 고정**(양측, 색별 미러) 및 정당화.
- **사전등록**: prior, pooled vs hierarchical, CrI 수준, "균형(0.5)" 판정 기준 — 본 run 전 확정.
- **동기 재서술**을 문서 전반에 반영(핸디캡 공정성 → 집중 vs 분산 직접 듀얼).
- (선택) **B2 확장:** NoQ + 모든 Q-variant를 라운드로빈으로 돌려 30점 배분 전체 랭킹. 게임 수↑.
